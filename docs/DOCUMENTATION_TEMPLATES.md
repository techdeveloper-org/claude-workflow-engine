## Implementation Rules

### Create Logic (File Doesn't Exist)
1. Scan entire codebase to understand:
   - Project structure & organization
   - Primary language(s) & frameworks
   - Key components & their responsibilities
   - Architecture patterns
   - Database/data models
   - External integrations
   - Technology stack

2. Fill ALL template placeholders with accurate info
3. Auto-detect from code:
   - `{PROJECT_NAME}`: From git repo or folder name
   - `{Languages}`: From file extensions
   - `{STATUS}`: From git tags or VERSION file
   - `{VERSION}`: From version.txt or package.json
   - `{Test command}`: From pytest.ini, package.json, etc.

4. Generate complete, coherent documentation

### Update Logic (File Exists)
1. Parse existing file to extract:
   - What's already documented
   - What needs updating
   - What's outdated

2. Identify changes from modified files:
   - New files = new sections
   - Modified files = section updates
   - Deleted files = remove references
   - New imports = update dependencies

3. Merge new content:
   - Keep existing structure
   - Add new sections
   - Update changed sections
   - Preserve manual notes/customizations

4. Preserve user customizations:
   - Don't overwrite custom sections
   - Keep "Contact & Support" sections as-is
   - Preserve any special notes
