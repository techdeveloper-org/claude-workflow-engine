# v1.18.0 — Runtime Verification

**Status:** PLANNED
**Target Branch:** `feature/v1.18.0-runtime-verification`
**Base:** `main` @ v1.17.0

---

## Overview

Runtime Verification adds a non-invasive contract-checking layer to every pipeline node.
When `ENABLE_RUNTIME_VERIFICATION=1`, the verifier checks FlowState preconditions before
each node runs and postconditions after, accumulating violations into a structured
`verification_report` in FlowState. The pipeline continues on violation (non-strict mode)
or halts on the first CRITICAL violation (`STRICT_RUNTIME_VERIFICATION=1`).

**Design principles:**
- Zero changes to existing node function signatures
- Decorator-based injection (`@verify_node(contract)`)
- Opt-in only — `ENABLE_RUNTIME_VERIFICATION=0` default (pipeline 100% unchanged when disabled)
- No I/O or LLM calls inside the verifier (< 5ms overhead per node)
- NullVerifier pattern: disabled state returns a no-op object, not `None` checks everywhere

### Phase A — Architecture & Blueprint

**Gate:** consensus-agent APPROVED before any code is written.

#### A-1: Design NodeContract DSL
- Define `NodeContract` dataclass: `node_name`, `preconditions`, `postconditions`, `invariants`
- Define `PreconditionSpec`: key name, expected type, required flag, optional min/max value
- Define `PostconditionSpec`: key name, non-null flag, min length
- Define `InvariantSpec`: description string + callable `check_fn(state) -> bool`
- Write 2 concrete example contracts:
  - `orchestration_pre_analysis_node` contract
  - `prompt_gen_expert_caller` contract
- **File:** `langgraph_engine/runtime_verification/contracts.py`

#### A-2: Design RuntimeVerifier + NullVerifier
- `RuntimeVerifier._registry: Dict[str, NodeContract]` — O(1) lookup by node name
- `RuntimeVerifier._violations: List[Violation]` — append-only accumulator
- `RuntimeVerifier.get_instance()` — returns `NullVerifier` when env flag is off
- `NullVerifier` — all methods return empty list / None (Null Object Pattern)
- Registry Pattern: contracts registered at import time via `verifier.register(contract)`
- **File:** `langgraph_engine/runtime_verification/verifier.py`

#### A-3: Design `@verify_node` Decorator
- Decorator wraps existing node `fn(state) -> dict` without changing its signature
- Pre-call: `verifier.check_preconditions(contract.node_name, state)`
- Post-call: `verifier.check_postconditions(contract.node_name, result)`
- On violation: log via `core/get_logger` — never raise in non-strict mode
- **File:** `langgraph_engine/runtime_verification/decorators.py`

#### A-4: Design Level Transition Guards
- Guard table: `Dict[Tuple[str,str], List[PreconditionSpec]]`
- Guards required:

  | From | To | Required Keys | Constraints |
  |---|---|---|---|
  | `level_minus1` | `level1` | `auto_fix_complete` | `bool = True` |
  | `level1` | `level3` | `combined_complexity_score`, `session_synced` | score in [1,25]; synced = True |
  | `pre_analysis` | `step0` | `pre_analysis_result`, `call_graph_metrics` | both non-null dict |
  | `step0` | `step8` | `orchestration_prompt`, `orchestrator_result` | len > 200, len > 50 |

- **File:** `langgraph_engine/runtime_verification/invariants.py`

#### A-5: Design VerificationReport
- `@dataclass VerificationReport`: `verified_nodes`, `violations`, `warnings`, `pass_fail`, `elapsed_ms_per_node`
- `to_dict()` method — JSON-serializable (used when writing to FlowState)
- **File:** `langgraph_engine/runtime_verification/report.py`

#### A-6: Architecture Decision Records
- **ADR-003:** Why decorator pattern over subclassing node functions
- **ADR-004:** Why opt-in default (`ENABLE_RUNTIME_VERIFICATION=0`)
- **ADR-005:** Why no LLM calls in verifier (latency contract)
- **File:** `docs/adr/ADR-003-runtime-verification-decorator.md`, etc.

### Phase B — Core Implementation

**Gate:** All files importable with zero circular imports; `python -c "import langgraph_engine.runtime_verification"` exits 0.

**Sequence:** implement files in this exact order to avoid forward-reference issues.

#### B-1: `contracts.py`
- `PreconditionSpec` dataclass
- `PostconditionSpec` dataclass
- `InvariantSpec` dataclass
- `Violation` dataclass: `node_name`, `check_type`, `key`, `message`, `severity` (INFO/WARNING/ERROR/CRITICAL)
- `NodeContract` dataclass (composes above)
- No imports from other `runtime_verification/` files — pure dataclasses only

#### B-2: `report.py`
- `VerificationReport` dataclass
- `to_dict()` returning `dict` with all fields JSON-serializable
- Imports only: `dataclasses`, `typing` — no internal imports

#### B-3: `verifier.py`
- `NullVerifier` class — all public methods are no-ops
- `RuntimeVerifier` class:
  - `_registry: Dict[str, NodeContract]`
  - `_violations: List[Violation]`
  - `_verified_nodes: List[str]`
  - `_elapsed: Dict[str, float]`
  - `get_instance() -> Union[RuntimeVerifier, NullVerifier]` — reads `ENABLE_RUNTIME_VERIFICATION` env var
  - `register(contract: NodeContract)` — adds to registry
  - `check_preconditions(node_name, state) -> List[Violation]`
  - `check_postconditions(node_name, state) -> List[Violation]`
  - `check_level_transition(from_level, to_level, state) -> List[Violation]`
  - `build_report() -> VerificationReport`
- Imports: `contracts.py`, `report.py`, `core/logger_factory.py`

#### B-4: `decorators.py`
- `verify_node(contract: NodeContract)` decorator factory
- Inner `wrapper(state, *args, **kwargs)`:
  1. Get verifier instance (`RuntimeVerifier.get_instance()`)
  2. `check_preconditions(contract.node_name, state)`
  3. Call original `fn(state, *args, **kwargs)` — always called, never skipped
  4. `check_postconditions(contract.node_name, result)` — if result is dict
  5. Append violations to verifier accumulator
  6. Log summary via `core/get_logger`
  7. Return original `result` unchanged
- `functools.wraps(fn)` applied — preserves function metadata
- Imports: `verifier.py`, `contracts.py`, `functools`, `os`

#### B-5: `invariants.py`
- `LEVEL_TRANSITION_GUARDS: Dict[Tuple[str, str], List[PreconditionSpec]]`
- `BUILTIN_INVARIANTS: List[InvariantSpec]` — e.g., complexity score range assertion
- `def get_transition_guard(from_level: str, to_level: str) -> List[PreconditionSpec]`
- Imports: `contracts.py` only

#### B-6: `schema_verifier.py`
- `verify_orchestration_prompt(prompt: str) -> List[str]`
  - Check: non-empty, `len(prompt) > 200`, `"Phase" in prompt`
  - Returns list of error strings (empty = valid)
- `verify_orchestrator_result(result: str) -> List[str]`
  - Check: non-empty, `len(result) > 50`, not purely an error message
  - Returns list of error strings
- No internal imports — pure string functions

#### B-7: `__init__.py`
- Exports: `RuntimeVerifier`, `NullVerifier`, `NodeContract`, `verify_node`, `VerificationReport`, `Violation`
- Last file created (after all others are stable)

#### B-8: FlowState Extension
- **File:** `langgraph_engine/state/state_definition.py`
  - Add `verification_report: Optional[Dict]` with default `None`
  - Add `verification_violations: List[str]` with default `[]`
- **File:** `langgraph_engine/state/reducers.py`
  - Add reducer for `verification_report` — last-write-wins
  - Add reducer for `verification_violations` — append (same as other list state keys)

#### B-9: Routing Guard Integration
- **File:** `langgraph_engine/routing/` — identify the level transition routing functions
- In each routing function that emits a level edge, add:
  ```python
  verifier = RuntimeVerifier.get_instance()
  violations = verifier.check_level_transition(from_level, to_level, state)
  # log violations; if STRICT_RUNTIME_VERIFICATION=1 raise RoutingError
  ```
- Do NOT change routing return values — guard is observational by default

#### B-10: QualityGate Extension
- **File:** `langgraph_engine/quality_gate.py`
- Add Gate 5 — `verification_gate`:
  - Skip entirely if `ENABLE_RUNTIME_VERIFICATION != "1"`
  - Read `state.get("verification_report")`
  - If `None` → pass with warning "no report found"
  - If `pass_fail=False` and `STRICT_RUNTIME_VERIFICATION=1` → raise `QualityGateError`
  - If `pass_fail=False` and strict=0 → log warning, return `(False, "N violations")`
  - If `pass_fail=True` → return `(True, "verification clean")`

#### B-11: Configuration & Documentation Updates

  ```
  STRICT_RUNTIME_VERIFICATION=0
  VERIFICATION_LOG_LEVEL=WARNING
  ```

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `ENABLE_RUNTIME_VERIFICATION` | `0` | Enable runtime contract/invariant verification. `0` = pipeline unchanged. |
| `STRICT_RUNTIME_VERIFICATION` | `0` | If `1`, pipeline halts on first CRITICAL violation. |
| `VERIFICATION_LOG_LEVEL` | `WARNING` | Log verbosity for verifier output. |

---

## Non-Goals (explicitly out of scope for v1.18.0)

- Formal mathematical proof of pipeline correctness (static analysis only — not runtime)
- LLM-assisted violation diagnosis (verifier is pure Python, no LLM calls)
- Retroactive verification of historical pipeline runs
- Verification of MCP server tool calls (MCP boundary is separate concern)
- Modifying any existing node function in `level3_execution/nodes/`
