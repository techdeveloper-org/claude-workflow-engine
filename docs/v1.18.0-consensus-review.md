## Open Question Resolutions

**Q1 — Per-node vs global `verification_violations` list?**
RESOLUTION: **Keep global list (current design).**
`Violation.node_name` field already enables post-hoc grouping. Nested structure deferred to v1.19.0 after shadow-mode data reveals real violation patterns.

**Q2 — Transition guards fire BEFORE or AFTER routing decision?**
RESOLUTION: **BEFORE (current design), confirmed.**
Guards are preconditions on the state required for a level to execute. Checking after routing is semantically incorrect — violation would be logged after the irreversible decision. Aligns with fail-fast principle.

**Q3 — Expose synchronous health probe `/health/verification`?**
RESOLUTION: **DEFER to v1.19.0.**
Adding HTTP surface to runtime_verification/ violates ADR-005 (no network in verifier). Extend existing `scripts/health_server.py` `/health` endpoint instead.

**Q4 — `NODE_CONTRACTS` loadable from YAML?**
RESOLUTION: **NO for v1.18.0.**
YAML loading introduces file I/O in the startup path, contradicting the no-I/O principle. Contracts live in version control as Python code. Defer YAML-based contract authoring CLI to a future release.

## Non-Blocking Observations

1. **FlowState type annotation mismatch** — `verification_violations: List[Dict[str, Any]]` in TypedDict but decorator appends `Violation` dataclass instances. Implementer must use `asdict(v)` before appending to keep the field JSON-serialisable. Must be resolved during B-4 (decorators.py implementation).

2. **Silent miss on unregistered node** — `contracts.get(node_name)` returns `None` silently when a node has no contract. Add `_LOG.warning("[RuntimeVerifier] no contract registered for node: %s", node_name)` in `check_preconditions` so engineers are notified when they add a node without a contract.

3. **Explicit test plan missing** — 42 implied test cases must be inferred by the implementer. An explicit `## Test Coverage` section in the blueprint listing all 42 as a checklist would reduce ambiguity for qa-testing-agent.

4. **Routing function names not enumerated** — Section 5.1 shows `@verify_transition` usage but does not name which routing functions in `langgraph_engine/routing/` receive it. Implementer must inspect the routing package to identify which functions cross level boundaries. List them explicitly to prevent missed wrappings.

5. **Singleton not thread-safe** — `get_instance()` check-then-set is non-atomic. Low priority while the pipeline is single-threaded. Add `threading.Lock()` guard if the engine moves to threaded execution.

## Conditions

None. The type annotation mismatch (Observation 1) is a blueprint-level ambiguity to be resolved during implementation — not an architectural blocker requiring architect revision.
