# v1.18.0 — Runtime Verification Blueprint

**Status:** Draft (solution-architect) → pending consensus-agent review
**Date:** 2026-04-14
**Author:** solution-architect
**Feature:** Runtime Verification Layer for LangGraph Pipeline

---

## Executive Summary

Runtime Verification (RV) adds an additive, decorator-based contract-enforcement
layer on top of existing pipeline nodes. It introduces **no changes** to node
function bodies, no changes to the LangGraph StateGraph topology, and no changes
to existing routing. The verifier is off by default (`ENABLE_RUNTIME_VERIFICATION=0`)
and, when enabled, intercepts node execution via a decorator to validate
preconditions, postconditions, and invariants declared in a separate contract
module.

**Critical constraint (non-negotiable):** This feature **MUST NOT rewrite any
existing node function**. Verification is attached externally via `@verify_node`
at node-registration time or through a post-hoc wrap utility.

**Latency budget:** The verifier MUST add ≤ 5ms per node call when enabled.
**No LLM calls.** **No I/O.** **No network.** Verification is pure in-memory
predicate evaluation.

---

## 1. Package Structure

A new package `langgraph_engine/runtime_verification/` contains all verification
logic. It is a sibling to `langgraph_engine/core/` and `langgraph_engine/level3_execution/`.

```
langgraph_engine/runtime_verification/
├── __init__.py              # Public exports: verify_node, RuntimeVerifier, NodeContract
├── contracts.py             # DSL dataclasses: NodeContract, PreconditionSpec,
│                            #   PostconditionSpec, InvariantSpec, Violation
├── verifier.py              # RuntimeVerifier (real) + NullVerifier (no-op)
│                            #   singleton via get_instance()
├── decorators.py            # @verify_node decorator (wraps node callables)
├── invariants.py            # Common reusable invariant check functions
│                            #   (non_null, type_check, range_check, len_min,
│                            #    dict_has_keys, is_positive, is_oneof)
├── schema_verifier.py       # Schema validation helpers for dict/TypedDict
├── report.py                # VerificationReport dataclass + aggregator
└── registry.py              # NODE_CONTRACTS: Dict[str, NodeContract]
                             # LEVEL_TRANSITION_GUARDS: Dict[Tuple[str,str], ...]
```

**Zero external dependencies.** Uses only stdlib (`dataclasses`, `typing`,
`functools`, `time`, `os`, `logging`).

**Integration points (read-only imports from existing engine):**
- `langgraph_engine.core.logger_factory.get_logger` — structured logging
- `langgraph_engine.core.error_handler` — error classification (optional)
- `langgraph_engine.state.flow_state.FlowState` — TypedDict reference

**No reverse imports:** existing engine modules do **not** import
`runtime_verification` unless they are the decorator wiring layer.

## 2. NodeContract DSL

The DSL is a set of frozen dataclasses describing what a node expects from and
promises to `FlowState`. Contracts are **declarative**: they contain specs, not
executable logic. The verifier interprets them.

### 2.1 Dataclass definitions

```python
# langgraph_engine/runtime_verification/contracts.py
"""
NodeContract DSL — declarative specifications for pipeline node contracts.

A NodeContract describes:
  - preconditions  : what FlowState fields MUST exist and be valid before the node runs
  - postconditions : what FlowState fields MUST exist and be valid after the node runs
  - invariants     : what FlowState-wide properties MUST hold during the node's execution
  - tags           : free-form labels for filtering (level, feature, priority)

Contracts are pure data. They contain no executable logic. The verifier
(RuntimeVerifier) interprets them against the FlowState snapshot.
"""

from __future__ import annotations

from dataclasses import dataclass, field
from typing import Any, Callable, List, Optional, Tuple, Type, Union


# ------------------------------------------------------------------------------
# Spec primitives
# ------------------------------------------------------------------------------

@dataclass(frozen=True)
class PreconditionSpec:
    """
    A single precondition on a FlowState field.

    Attributes:
        field_name    : name of the FlowState key to check.
        expected_type : Python type or tuple of types; None to skip type check.
        required      : if True, field MUST be present and non-None.
        min_val       : for numeric / len-checkable fields, minimum allowed value.
                        For str/list/dict, this is minimum LENGTH.
        max_val       : for numeric / len-checkable fields, maximum allowed value.
                        For str/list/dict, this is maximum LENGTH.
        allowed_values: if set, value MUST be in this iterable.
        custom_check  : optional callable (value) -> (bool, str). Returns
                        (True, "") on pass or (False, reason) on fail.
        message       : override default violation message.
    """
    field_name: str
    expected_type: Optional[Union[Type, Tuple[Type, ...]]] = None
    required: bool = True
    min_val: Optional[Union[int, float]] = None
    max_val: Optional[Union[int, float]] = None
    allowed_values: Optional[Tuple[Any, ...]] = None
    custom_check: Optional[Callable[[Any], Tuple[bool, str]]] = None
    message: Optional[str] = None


@dataclass(frozen=True)
class PostconditionSpec:
    """
    A single postcondition on a FlowState field (checked AFTER node execution).

    Same semantics as PreconditionSpec. Postconditions are checked against the
    node's return value merged into FlowState.
    """
    field_name: str
    expected_type: Optional[Union[Type, Tuple[Type, ...]]] = None
    required: bool = True
    min_val: Optional[Union[int, float]] = None
    max_val: Optional[Union[int, float]] = None
    allowed_values: Optional[Tuple[Any, ...]] = None
    custom_check: Optional[Callable[[Any], Tuple[bool, str]]] = None
    message: Optional[str] = None


@dataclass(frozen=True)
class InvariantSpec:
    """
    A FlowState-wide invariant that must hold during node execution.

    Unlike pre/postconditions, invariants are not tied to a single field.
    They receive the entire FlowState and return (ok, reason).

    Attributes:
        name         : human-readable label for the invariant.
        check        : callable (state: dict) -> (bool, str).
        severity     : 'error' | 'warning'. Warnings do not fail gate 5.
        message      : override default violation message.
    """
    name: str
    check: Callable[[dict], Tuple[bool, str]]
    severity: str = "error"
    message: Optional[str] = None


# ------------------------------------------------------------------------------
# NodeContract
# ------------------------------------------------------------------------------

@dataclass(frozen=True)
class NodeContract:
    """
    Declarative contract for a pipeline node.

    Attributes:
        node_name      : name used for @verify_node lookup and reporting.
        preconditions  : list of PreconditionSpec checked before node execution.
        postconditions : list of PostconditionSpec checked after node execution.
        invariants     : list of InvariantSpec checked during execution.
        tags           : free-form labels (e.g., ('level3', 'step0', 'critical')).
        latency_budget_ms : soft warning threshold — verifier logs if node exceeds.
        description    : human-readable explanation of what the node guarantees.
    """
    node_name: str
    preconditions: Tuple[PreconditionSpec, ...] = field(default_factory=tuple)
    postconditions: Tuple[PostconditionSpec, ...] = field(default_factory=tuple)
    invariants: Tuple[InvariantSpec, ...] = field(default_factory=tuple)
    tags: Tuple[str, ...] = field(default_factory=tuple)
    latency_budget_ms: Optional[int] = None
    description: str = ""


# ------------------------------------------------------------------------------
# Violation record
# ------------------------------------------------------------------------------

@dataclass(frozen=True)
class Violation:
    """
    A single contract violation detected by the verifier.

    Attributes:
        node_name   : name of the node whose contract was violated.
        phase       : 'pre' | 'post' | 'invariant' | 'transition'.
        field_name  : FlowState field involved (empty for invariants).
        reason      : human-readable explanation.
        severity    : 'error' | 'warning'.
        timestamp   : epoch seconds when violation occurred.
    """
    node_name: str
    phase: str
    field_name: str
    reason: str
    severity: str = "error"
    timestamp: float = 0.0
```

```python
            message="pre_analysis_result must be a non-empty dict",

        PostconditionSpec(

            required=True,
            min_val=1,

    invariants=(
        InvariantSpec(
            name="complexity_score_unchanged",

                False, "complexity score disappeared during pre-analysis"

            severity="error",
        ),
    ),
    tags=("level3", "pre-0", "critical"),

)
```

## 3. Decorator Pattern (`@verify_node`)

### 3.1 Decorator design

The decorator is the single integration point with existing nodes. It is applied
**at the wrapping layer**, not inside node function bodies.

```python
# langgraph_engine/runtime_verification/decorators.py
"""
@verify_node — decorator that wraps a LangGraph node callable with runtime
              verification checks. Non-invasive: does NOT modify the wrapped
              function; returns a thin wrapper that delegates to it.

Integration:
    Nodes are registered with the decorator either at import time OR at
    StateGraph build time via a post-hoc wrap utility. No existing node
    function body is touched.

Usage:
    from langgraph_engine.runtime_verification import verify_node

    @verify_node("orchestration_pre_analysis_node")
    def orchestration_pre_analysis_node(state: FlowState) -> dict:
        ...

    OR  (post-hoc wrap at build time):

    wrapped = verify_node("step0")(step0_node)
    graph.add_node("step0", wrapped)
"""

from __future__ import annotations

import functools
import logging
import time
from typing import Any, Callable, Dict

from .verifier import RuntimeVerifier

_LOG = logging.getLogger(__name__)


def verify_node(node_name: str) -> Callable:
    """
    Wrap a LangGraph node callable with runtime verification.

    When ENABLE_RUNTIME_VERIFICATION=0 (default), the decorator returns the
    ORIGINAL function unchanged — zero runtime overhead.

    When ENABLE_RUNTIME_VERIFICATION=1, the decorator returns a wrapper that:
      1. Looks up the NodeContract by node_name.
      2. Verifies preconditions against the input state snapshot.
      3. Calls the original node.
      4. Merges the return value into a post-state snapshot.
      5. Verifies postconditions and invariants.
      6. Appends violations to state['verification_violations'].
      7. Returns the original return value unchanged.

    The wrapper NEVER modifies the wrapped function's return value beyond
    appending to 'verification_violations' (which the node does not read).
    """
    def decorator(fn: Callable[[Dict[str, Any]], Any]) -> Callable:
        verifier = RuntimeVerifier.get_instance()

        # Null-verifier fast path: return original function, zero overhead.
        if not verifier.enabled:
            return fn

        @functools.wraps(fn)
        def wrapper(state: Dict[str, Any]) -> Any:
            t0 = time.perf_counter()

            # --- PRE phase --------------------------------------------------
            pre_violations = verifier.check_preconditions(node_name, state)

            # --- EXECUTE (delegate to original node) -----------------------
            result = fn(state)

            # --- POST phase -------------------------------------------------
            # Build a post-state snapshot. LangGraph nodes return a dict of
            # state updates; we merge shallow-ly into a copy of the input
            # state (do NOT mutate the caller's state object).
            post_state = dict(state)
            if isinstance(result, dict):
                post_state.update(result)

            post_violations = verifier.check_postconditions(node_name, post_state)
            inv_violations = verifier.check_invariants(node_name, post_state)

            # --- Aggregate -------------------------------------------------
            all_v = list(pre_violations) + list(post_violations) + list(inv_violations)
            if all_v:
                existing = state.get("verification_violations") or []
                # Only write back into the caller's result dict, not input state
                if isinstance(result, dict):
                    result.setdefault("verification_violations", list(existing))
                    result["verification_violations"].extend(all_v)
                else:
                    _LOG.warning(
                        "[verify_node:%s] %d violations but node returned non-dict; "
                        "dropping violations (node must return dict for RV to work)",
                        node_name, len(all_v),
                    )

            # --- Latency logging (soft) ------------------------------------
            elapsed_ms = (time.perf_counter() - t0) * 1000.0
            verifier.record_latency(node_name, elapsed_ms)

            return result

        wrapper.__rv_wrapped__ = True  # marker for introspection / tests
        return wrapper

    return decorator
```

### 3.2 The < 5ms guarantee

The decorator enforces the latency budget through **capability constraints**, not
through time-based circuit breakers:

1. **No I/O** — the verifier is a pure in-memory predicate evaluator. It never
   reads files, never calls network services, never spawns subprocesses.
2. **No LLM calls** — enforced architecturally by not importing any LLM client.
3. **Specs are frozen dataclasses** — lookup is O(1) dict access, no parsing.
4. **Predicates are tiny** — `isinstance`, comparison, `in` checks, `len()`.
5. **Shallow state copy only** — `dict(state)` is O(n) in keys (~30 keys typical),
   which is sub-millisecond.
6. **No recursion into nested structures** — verification checks top-level
   FlowState keys. Deep structural validation is opt-in via `custom_check`.

**Empirical target (to be validated in Phase 2 tests):**
- Typical node with 3 pre + 2 post + 1 invariant: ~0.3-0.8 ms overhead.
- Worst case (10 pre + 10 post + 5 invariants): ~2-3 ms.
- **Hard cap:** if `record_latency` ever sees > 5ms for a single node, the verifier
  logs a warning and increments a metric counter, but does NOT raise. The
  opt-in nature means users tolerate degraded latency only when they explicitly
  enable verification.

## 4. RuntimeVerifier + NullVerifier (Singleton)

### 4.1 Design

```python
# langgraph_engine/runtime_verification/verifier.py
"""
RuntimeVerifier — singleton that interprets NodeContract specs against FlowState.

Two implementations share the RuntimeVerifier public interface:
  - RuntimeVerifier : real checks. Active when ENABLE_RUNTIME_VERIFICATION=1.
  - NullVerifier    : no-op fast path. Active when ENABLE_RUNTIME_VERIFICATION=0.

Both are selected by `get_instance()` at first access. The choice is cached
for the process lifetime; toggling the env var mid-run has no effect.
"""

from __future__ import annotations

import logging
import os
import time
from typing import Any, Dict, List, Optional

from .contracts import (
    NodeContract, PreconditionSpec, PostconditionSpec, InvariantSpec, Violation
)

_LOG = logging.getLogger(__name__)


class _VerifierBase:
    """Interface shared by RuntimeVerifier and NullVerifier."""
    enabled: bool = False

    def check_preconditions(self, node_name: str, state: Dict[str, Any]) -> List[Violation]:
        return []

    def check_postconditions(self, node_name: str, state: Dict[str, Any]) -> List[Violation]:
        return []

    def check_invariants(self, node_name: str, state: Dict[str, Any]) -> List[Violation]:
        return []

    def check_transition(self, from_level: str, to_level: str, state: Dict[str, Any]) -> List[Violation]:
        return []

    def record_latency(self, node_name: str, ms: float) -> None:
        pass

    def register_contract(self, contract: NodeContract) -> None:
        pass

    def get_report(self) -> Dict[str, Any]:
        return {"pass_fail": True, "violations": [], "latency_stats": {}}


class NullVerifier(_VerifierBase):
    """No-op verifier. Used when ENABLE_RUNTIME_VERIFICATION is not set."""
    enabled = False


class RuntimeVerifier(_VerifierBase):
    """Real verifier. Interprets NodeContract specs."""
    enabled = True

    _instance: Optional[_VerifierBase] = None

    def __init__(self) -> None:
        from .registry import NODE_CONTRACTS, LEVEL_TRANSITION_GUARDS
        self._contracts: Dict[str, NodeContract] = dict(NODE_CONTRACTS)
        self._level_guards = dict(LEVEL_TRANSITION_GUARDS)
        self._violations: List[Violation] = []
        self._latency_ms: Dict[str, List[float]] = {}

    # -- Factory ---------------------------------------------------------

    @classmethod
    def get_instance(cls) -> "_VerifierBase":
        if cls._instance is None:
            if os.getenv("ENABLE_RUNTIME_VERIFICATION", "0") == "1":
                cls._instance = RuntimeVerifier()
                _LOG.info("[RuntimeVerifier] enabled — %d contracts loaded",
                          len(cls._instance._contracts))
            else:
                cls._instance = NullVerifier()
        return cls._instance

    @classmethod
    def reset_for_tests(cls) -> None:
        """Test-only: clear the singleton so tests can toggle env vars."""
        cls._instance = None

    # -- Contract registration ------------------------------------------

    def register_contract(self, contract: NodeContract) -> None:
        self._contracts[contract.node_name] = contract

    # -- Check methods --------------------------------------------------

    def check_preconditions(self, node_name: str, state: Dict[str, Any]) -> List[Violation]:
        c = self._contracts.get(node_name)
        if c is None:
            return []
        return [v for v in (
            self._check_spec(spec, state, node_name, "pre")
            for spec in c.preconditions
        ) if v is not None]

    def check_postconditions(self, node_name: str, state: Dict[str, Any]) -> List[Violation]:
        c = self._contracts.get(node_name)
        if c is None:
            return []
        return [v for v in (
            self._check_spec(spec, state, node_name, "post")
            for spec in c.postconditions
        ) if v is not None]

    def check_invariants(self, node_name: str, state: Dict[str, Any]) -> List[Violation]:
        c = self._contracts.get(node_name)
        if c is None:
            return []
        out: List[Violation] = []
        for inv in c.invariants:
            try:
                ok, reason = inv.check(state)
            except Exception as e:
                ok, reason = False, f"invariant raised: {e}"
            if not ok:
                out.append(Violation(
                    node_name=node_name, phase="invariant",
                    field_name="", reason=inv.message or reason,
                    severity=inv.severity, timestamp=time.time(),
                ))
        self._violations.extend(out)
        return out

    def check_transition(self, from_level: str, to_level: str,
                         state: Dict[str, Any]) -> List[Violation]:
        specs = self._level_guards.get((from_level, to_level), [])
        out: List[Violation] = []
        for spec in specs:
            v = self._check_spec(spec, state,
                                 f"transition:{from_level}->{to_level}", "transition")
            if v is not None:
                out.append(v)
        return out

    # -- Internal: single-spec evaluator --------------------------------

    def _check_spec(self, spec, state: Dict[str, Any],
                    node_name: str, phase: str) -> Optional[Violation]:
        """Evaluate a single Pre/PostconditionSpec. Returns Violation or None."""
        val = state.get(spec.field_name)

        # Required / None check
        if spec.required and val is None:
            return self._violation(node_name, phase, spec.field_name,
                                   spec.message or f"{spec.field_name} is required but missing")

        if val is None:
            return None  # optional field absent — OK

        # Type check
        if spec.expected_type is not None and not isinstance(val, spec.expected_type):
            return self._violation(node_name, phase, spec.field_name,
                                   spec.message or
                                   f"{spec.field_name}: expected {spec.expected_type}, got {type(val).__name__}")

        # Range / length check
        if spec.min_val is not None or spec.max_val is not None:
            comparable = len(val) if isinstance(val, (str, list, tuple, dict, set)) else val
            if spec.min_val is not None and comparable < spec.min_val:
                return self._violation(node_name, phase, spec.field_name,
                                       spec.message or f"{spec.field_name}: below min ({comparable} < {spec.min_val})")
            if spec.max_val is not None and comparable > spec.max_val:
                return self._violation(node_name, phase, spec.field_name,
                                       spec.message or f"{spec.field_name}: above max ({comparable} > {spec.max_val})")

        # Allowed values
        if spec.allowed_values is not None and val not in spec.allowed_values:
            return self._violation(node_name, phase, spec.field_name,
                                   spec.message or f"{spec.field_name}: {val!r} not in allowed set")

        # Custom check
        if spec.custom_check is not None:
            try:
                ok, reason = spec.custom_check(val)
            except Exception as e:
                ok, reason = False, f"custom_check raised: {e}"
            if not ok:
                return self._violation(node_name, phase, spec.field_name,
                                       spec.message or reason)

        return None

    def _violation(self, node_name: str, phase: str, field_name: str, reason: str) -> Violation:
        v = Violation(
            node_name=node_name, phase=phase,
            field_name=field_name, reason=reason,
            severity="error", timestamp=time.time(),
        )
        self._violations.append(v)
        return v

    # -- Latency tracking -----------------------------------------------

    def record_latency(self, node_name: str, ms: float) -> None:
        self._latency_ms.setdefault(node_name, []).append(ms)
        c = self._contracts.get(node_name)
        if c is not None and c.latency_budget_ms is not None and ms > c.latency_budget_ms:
            _LOG.warning("[RuntimeVerifier] %s exceeded latency budget: %.1fms > %dms",
                         node_name, ms, c.latency_budget_ms)
        if ms > 5.0:
            _LOG.warning("[RuntimeVerifier] verification overhead for %s = %.2fms (>5ms target)",
                         node_name, ms)

    # -- Report ---------------------------------------------------------

    def get_report(self) -> Dict[str, Any]:
        return {
            "pass_fail": len([v for v in self._violations if v.severity == "error"]) == 0,
            "violations": [v.__dict__ for v in self._violations],
            "latency_stats": {
                name: {
                    "count": len(lst),
                    "avg_ms": sum(lst) / len(lst) if lst else 0.0,
                    "max_ms": max(lst) if lst else 0.0,
                }
                for name, lst in self._latency_ms.items()
            },
        }
```

### 4.2 Singleton rationale

- One verifier per process keeps the contract registry and violation log
  unified across nodes.
- `get_instance()` decides between `RuntimeVerifier` and `NullVerifier` ONCE at
  first access, based on `ENABLE_RUNTIME_VERIFICATION`. Subsequent calls return
  the same object — toggling the env var mid-run is **not** supported.
- `reset_for_tests()` allows test suites to re-initialise the singleton after
  monkey-patching environment variables.

## 5. LEVEL_TRANSITION_GUARDS

Level transition guards are contracts on **state boundaries** between LangGraph
sub-pipelines. They are not tied to a single node; they fire when a routing
function crosses from one level to the next.

```python
# langgraph_engine/runtime_verification/registry.py  (excerpt)

from typing import Dict, List, Tuple
from .contracts import PreconditionSpec

LEVEL_TRANSITION_GUARDS: Dict[Tuple[str, str], List[PreconditionSpec]] = {
    # Level -1 (auto-fix) → Level 1 (sync)
    ("level_minus1", "level1"): [
        PreconditionSpec(
            field_name="auto_fix_complete",
            expected_type=bool,
            required=True,
            message="Level -1 auto-fix must complete before Level 1 sync",
        ),
    ],

    # Level 1 (sync) → Level 3 (execution)
    ("level1", "level3"): [
        PreconditionSpec(
            field_name="combined_complexity_score",
```

### 5.1 How transition guards are invoked

Transition guards are invoked from routing functions under
`langgraph_engine/routing/`. A helper decorator wraps conditional-edge
functions:

```python
# langgraph_engine/runtime_verification/decorators.py  (excerpt)

def verify_transition(from_level: str, to_level: str) -> Callable:
    """
    Decorator for routing functions. When the route fires, evaluates
    LEVEL_TRANSITION_GUARDS[(from_level, to_level)] against the current state.
    """
    def decorator(fn: Callable[[Dict[str, Any]], str]) -> Callable:
        verifier = RuntimeVerifier.get_instance()
        if not verifier.enabled:
            return fn

        @functools.wraps(fn)
        def wrapper(state: Dict[str, Any]) -> str:
            violations = verifier.check_transition(from_level, to_level, state)
            if violations:
                _LOG.warning("[verify_transition] %s -> %s: %d violations",
                             from_level, to_level, len(violations))
            return fn(state)

        return wrapper
    return decorator
```

**Non-invasive:** routing functions are wrapped at the same point nodes are —
at StateGraph build time in `pipeline_builder.py`. No routing logic is rewritten.

## 6. VerificationReport

### 6.1 Dataclass

```python
# langgraph_engine/runtime_verification/report.py
"""
VerificationReport — aggregate of all violations and latency stats for a run.

Produced at pipeline end. Consumed by:
  - quality_gate.py  → Gate 5 decision
  - metrics_exporter → Prometheus counters
  - audit_logger     → JSON audit line
  - FlowState        → state['verification_report'] field
"""

from __future__ import annotations

from dataclasses import asdict, dataclass, field
from typing import Any, Dict, List

from .contracts import Violation


@dataclass
class VerificationReport:
    """Aggregate result of a single pipeline run's verification."""
    pass_fail: bool = True
    total_nodes_verified: int = 0
    total_violations: int = 0
    error_count: int = 0
    warning_count: int = 0
    violations: List[Violation] = field(default_factory=list)
    latency_stats: Dict[str, Dict[str, float]] = field(default_factory=dict)
    started_at: float = 0.0
    ended_at: float = 0.0

    def to_dict(self) -> Dict[str, Any]:
        return {
            "pass_fail": self.pass_fail,
            "total_nodes_verified": self.total_nodes_verified,
            "total_violations": self.total_violations,
            "error_count": self.error_count,
            "warning_count": self.warning_count,
            "violations": [asdict(v) for v in self.violations],
            "latency_stats": self.latency_stats,
            "started_at": self.started_at,
            "ended_at": self.ended_at,
        }

    @classmethod
    def from_verifier(cls, verifier) -> "VerificationReport":
        raw = verifier.get_report()
        errors = [v for v in verifier._violations if v.severity == "error"]
        warnings = [v for v in verifier._violations if v.severity == "warning"]
        return cls(
            pass_fail=len(errors) == 0,
            total_nodes_verified=len(verifier._latency_ms),
            total_violations=len(verifier._violations),
            error_count=len(errors),
            warning_count=len(warnings),
            violations=list(verifier._violations),
            latency_stats=raw["latency_stats"],
        )
```

## 8. Quality Gate 5 (Verification Gate)

### 8.1 Gate 5 specification

A new gate is added to `langgraph_engine/quality_gate.py`. It is **opt-in via
env var** and runs at the same phase as existing gates (post-implementation,
pre-merge).

```python
# langgraph_engine/quality_gate.py  (additive)

def verification_gate(state: Dict[str, Any]) -> Tuple[bool, str]:
    """
    Gate 5: Runtime Verification Gate.

    Behavior:
      - ENABLE_RUNTIME_VERIFICATION=0 (default): PASS — gate is a no-op.
      - ENABLE_RUNTIME_VERIFICATION=1 and no report: PASS with warning.
      - ENABLE_RUNTIME_VERIFICATION=1 and report.pass_fail=True: PASS.
      - ENABLE_RUNTIME_VERIFICATION=1 and report.pass_fail=False:
          * STRICT_RUNTIME_VERIFICATION=1: raise QualityGateError.
          * STRICT_RUNTIME_VERIFICATION=0 (default): PASS with warning log.
    """
    if os.getenv("ENABLE_RUNTIME_VERIFICATION", "0") != "1":
        return (True, "verification gate disabled")

    report = state.get("verification_report")
    if report is None:
        return (True, "no verification report — skipping")

    if not report.get("pass_fail", True):
        violations = report.get("violations", [])
        if os.getenv("STRICT_RUNTIME_VERIFICATION", "0") == "1":
            raise QualityGateError(
                f"Runtime verification failed: {len(violations)} violations"
            )
        logger.warning(
            "[QualityGate] verification_gate: %d violations present (non-strict)",
            len(violations),
        )
    return (report.get("pass_fail", True),
            f"verification: {len(report.get('violations', []))} violations")
```

### 8.2 Env var contract

| Variable | Default | Meaning |
|----------|---------|---------|
| `ENABLE_RUNTIME_VERIFICATION` | `0` | Turn RV on (`1`) or off (`0`). Decorator is a no-op when off. |
| `STRICT_RUNTIME_VERIFICATION` | `0` | When RV is on, fail Gate 5 on any error-severity violation. When off, log warning only. |

**Rollout strategy:** enable in shadow mode first (`ENABLE=1, STRICT=0`) for
2 release cycles. Only after violation rate stabilises near zero should
`STRICT=1` be considered for CI pipelines.

## 9. DSA Choices

| Component | Data Structure | Algorithm | Rationale |
|-----------|---------------|-----------|-----------|
| `NODE_CONTRACTS` registry | `Dict[str, NodeContract]` (HashMap) | O(1) lookup by node_name | Decorator path must be sub-ms; hash lookup is the fastest general-purpose registry |
| `LEVEL_TRANSITION_GUARDS` | `Dict[Tuple[str,str], List[PreconditionSpec]]` | O(1) lookup by (from, to) | Transitions are rare but must be instant; keyed by pair avoids scanning |
| `_violations` accumulator | `List[Violation]` | Amortised O(1) append | Violations are write-many, read-once (at report time) |
| `_latency_ms` per-node | `Dict[str, List[float]]` | O(1) append, O(n) aggregation | Simple list per node; aggregation only at report time |
| Spec evaluation | Frozen dataclass + isinstance | O(1) per check | `isinstance` is C-level fast; frozen dataclass has no attribute resolution surprises |
| Report generation | Single-pass aggregation | O(V + N) where V=violations, N=nodes | Done once per run at pipeline end; negligible |
| State snapshot | `dict(state)` shallow copy | O(k) where k = top-level keys | ~30 keys typical; measured <0.1ms on modern hardware |

**Complexity summary:**
- Per node: `O(P + Q + I)` where P = preconditions, Q = postconditions, I = invariants.
- Typical: P+Q+I ≤ 10 → ~10 isinstance checks + 10 field lookups = sub-millisecond.
- Per pipeline run: `O(N * (P+Q+I))` where N = number of nodes (~15) → ~150 checks total.

## 10. Design Patterns

| Pattern | Location | Problem Solved |
|---------|----------|----------------|
| **Decorator** | `@verify_node`, `@verify_transition` | Attach verification logic externally to existing nodes/routes without modifying their source. Zero-invasive integration. |
| **Null Object** | `NullVerifier` | Eliminate `if verifier is None` branches and `if enabled` checks on the hot path. Disabled mode returns the original function with zero overhead. |
| **Singleton** | `RuntimeVerifier.get_instance()` | Single contract registry and violation log per process; avoids threading contract state through every node. |
| **Registry** | `NODE_CONTRACTS` dict, `LEVEL_TRANSITION_GUARDS` dict | O(1) lookup of contracts by name without scanning a list; new contracts registered by import-time dict insertion or `register_contract()`. |
| **Strategy** | `PreconditionSpec.custom_check` | Domain-specific checks (e.g., regex, cross-field consistency) plug in without extending the core spec class. |
| **Chain of Responsibility** | Spec evaluation order (required → type → range → allowed → custom) | Short-circuit on first failure; each check is independent and ordered by cost. |
| **Builder (existing)** | `PipelineBuilder.add_level*()` | RV wiring happens inside `add_*` methods — builder centralises the decorator application so individual levels don't know about verification. |
| **Repository (existing)** | State-read access patterns | Verification reads state through the same shallow-copy pattern as existing nodes, avoiding coupling to FlowState internals. |

### Pattern deviation rule

Per the `solution-architect` blueprint supremacy rule: implementing engineers
MAY NOT deviate from the patterns above without written architect approval.
Specifically:
- DO NOT subclass nodes to add verification (decorator pattern only).
- DO NOT add verification logic inside node function bodies.
- DO NOT call the verifier from routing functions directly (use
  `@verify_transition`).
- DO NOT introduce new singletons — the single `RuntimeVerifier` instance is
  the only stateful verification component.

## Failure Mode Analysis

| Failure Mode | Probability | Impact | Mitigation |
|--------------|-------------|--------|------------|
| Contract drift (contract says field X, code produces field Y) | Medium | High (false positives) | Contracts live in same repo as nodes; code-review gate requires contract update whenever node signature changes |
| Decorator latency exceeds 5ms | Low | Medium (pipeline slowdown) | Hard-coded no-I/O constraint; per-node latency warning at 5ms; metrics exporter tracks p99 |
| Verifier raises exception mid-node | Low | High (pipeline halt) | All `custom_check` and `invariant.check` calls wrapped in try/except; verifier never re-raises |
| NullVerifier path accidentally does work | Very Low | Medium (hidden overhead) | `decorator()` returns the original `fn` object directly when `enabled=False`; unit test asserts `wrapped is fn` |
| RV enabled in production without testing | Medium | Low (warnings only) | `STRICT_RUNTIME_VERIFICATION=0` default means violations never fail the pipeline in non-strict mode |
| Singleton state leaks between tests | Low | Medium (test flakiness) | `reset_for_tests()` classmethod; pytest fixture in `conftest.py` calls it before each test |
| FlowState field named `verification_violations` collides with future field | Very Low | High (data loss) | Namespace reserved; documented in flow_state.py docstring |

## Security Threat Model (STRIDE)

| Threat | Asset | Mitigation |
|--------|-------|------------|
| **S**poofing — malicious contract registered at runtime | Contract registry | Contracts loaded only from `registry.py`; `register_contract()` not exposed via network API |
| **T**ampering — verifier violations silently swallowed | VerificationReport | Audit logger writes report to append-only JSON log with daily rotation |
| **R**epudiation — "we never saw that violation" | Audit trail | Every violation includes `timestamp`, `node_name`, `reason` in structured log |
| **I**nformation disclosure — verification report leaks PII | Log sinks | Violation messages use field names only, never field values; task_description value is NEVER logged by RV |
| **D**enial of service — infinite invariant loop | Pipeline | Invariants wrapped in try/except; no verifier recursion; latency warning at 5ms |
| **E**levation of privilege — RV bypasses auth | N/A | RV is in-process and stateless; no external interface; no auth bypass surface |

## Open Questions for Consensus Review

1. **Should `verification_violations` be per-node or global?** Current design:
   global list appended by each decorator. Alternative: nested dict keyed by
   node name. Global is simpler; nested aids filtering but doubles complexity.
2. **Should transition guards fire BEFORE or AFTER the routing decision?**
   Current design: BEFORE — violations logged, but routing still happens.
   Alternative: AFTER — route result is verified against expected transitions.
3. **Should the verifier expose a synchronous health probe?** E.g.,
   `/health/verification` returning current violation count. Useful for
   Kubernetes readiness, but adds an HTTP surface area.
4. **Should NODE_CONTRACTS be loadable from YAML files** for non-Python
   extension by ops? Current design is Python-only; YAML adds parse cost but
   supports runtime contract updates.
