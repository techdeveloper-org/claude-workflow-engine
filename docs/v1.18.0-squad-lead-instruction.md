| `is_oneof(field_name, allowed)` | `(state: dict) -> (bool, str)` | `state.get(field_name) in allowed` |

Each function is a factory that returns the actual `(state: dict) -> (bool, str)` callable.
Example: `non_null("task_description")` returns a lambda/closure that checks the state dict.

Implementation notes for verifier.py:

1. `_VerifierBase` defines the interface. `NullVerifier` extends it with `enabled = False`
   and all methods returning empty lists or no-ops.

2. `RuntimeVerifier.get_instance()` is the only entry point for all consumers. The
   pattern:
   ```python
   @classmethod
   def get_instance(cls) -> "_VerifierBase":
       if cls._instance is None:
           if os.getenv("ENABLE_RUNTIME_VERIFICATION", "0") == "1":
               cls._instance = RuntimeVerifier()
           else:
               cls._instance = NullVerifier()
       return cls._instance
   ```

3. `reset_for_tests()` classmethod sets `cls._instance = None`. This is called in
   pytest conftest.py before each test that needs to toggle the env var.

4. NON-NEGOTIABLE from Consensus Observation 2: `check_preconditions` MUST emit a
   warning log when `self._contracts.get(node_name)` returns `None`:
   ```python
   def check_preconditions(self, node_name: str, state: dict) -> list:
       c = self._contracts.get(node_name)
       if c is None:
           _LOG.warning("[RuntimeVerifier] no contract registered for node: %s", node_name)
           return []
       ...
   ```
   Apply the same pattern to `check_postconditions` and `check_invariants`.

5. `_check_spec` evaluates specs in Chain of Responsibility order:
   required-check → type-check → range/len-check → allowed_values-check → custom_check.
   Return the first `Violation` found; return `None` if all pass.

6. `record_latency` logs a warning at two thresholds: the node's own
   `latency_budget_ms` (if set) AND the hard 5ms verifier overhead cap.

## 3. Non-Negotiables (Consensus Review — must confirm all 5 before marking complete)

These are not suggestions. Each must be present in the delivered code.

### N-1: asdict(v) before appending violations (Consensus Observation 1)

Location: `decorators.py` wrapper function.

Required pattern:
```python
from dataclasses import asdict
# ...
result["verification_violations"].extend(asdict(v) for v in all_v)
```

NOT acceptable:
```python
result["verification_violations"].extend(all_v)  # WRONG — appends Violation objects
```

Reason: `FlowState.verification_violations` is typed `List[Dict[str, Any]]`. Appending
`Violation` dataclass instances causes a type mismatch that breaks JSON serialisation
downstream (audit_logger, metrics_exporter).

### N-2: Warning log for unregistered node (Consensus Observation 2)

Location: `verifier.py`, `RuntimeVerifier.check_preconditions` (and the same pattern
in `check_postconditions` and `check_invariants`).

Required log line:
```python
_LOG.warning("[RuntimeVerifier] no contract registered for node: %s", node_name)
```

Reason: Without this, adding a new pipeline node without a contract silently passes
verification — giving a false sense of coverage. Engineers must be notified.

### N-3: Zero changes to existing node function signatures

No existing file in `langgraph_engine/level3_execution/nodes/`,
`langgraph_engine/level_minus1/`, or `langgraph_engine/level1_sync/` is modified.
The only existing files you touch are:

- `langgraph_engine/state/state_definition.py` (2 new Optional fields, append only)
- `langgraph_engine/level3_execution/quality_gate.py` (Gate 5 function added)
- `langgraph_engine/pipeline_builder.py` (verify_transition wiring in add_level* methods)
- `.env.example`, `CLAUDE.md`, `VERSION`, `CHANGELOG.md` (docs/config only)

### N-4: ENABLE_RUNTIME_VERIFICATION=0 default — zero pipeline change

Verification:

1. When `ENABLE_RUNTIME_VERIFICATION` is unset or `"0"`, `RuntimeVerifier.get_instance()`
   returns a `NullVerifier` instance.
2. `NullVerifier.enabled` is `False`.
3. `verify_node("any_node")(fn)` returns `fn` unchanged (the original function object,
   not a wrapper).
4. `verify_transition(...)` also returns the original routing function unchanged.
5. `verification_gate()` returns `(True, "verification gate disabled")` without reading state.

These five facts must be demonstrable by unit tests in `tests/test_runtime_verifier.py`.

### N-5: ASCII-only source files

All new files in `langgraph_engine/runtime_verification/` must contain ASCII characters
only. This is a project-wide requirement (Windows cp1252 safety). Check: no em-dashes,
no curly quotes, no Unicode bullets in comments or strings.

## 8. Routing Function Discovery (Consensus Observation 4 — action required)

The consensus review flags that blueprint Section 5.1 shows `@verify_transition`
usage but does not enumerate which routing functions to wrap. You must complete
this discovery as part of B-9.

Steps:
1. Read `langgraph_engine/routing/__init__.py` to see all exported routing functions.
2. Read `langgraph_engine/pipeline_builder.py` to find where each routing function
   is passed to `graph.add_conditional_edges(...)`.
3. Match each conditional edge call to the LEVEL_TRANSITION_GUARDS keys in registry.py:
   - `("level_minus1", "level1")` → wraps the routing function on the edge from
     level_minus1 to level1_session.
   - `("pre_analysis", "step0")` → wraps the routing function (or fixed edge) from
     the pre-analysis node to step0 inside the level3 subgraph.
   - `("step0", "step8")` → wraps the routing function (or fixed edge) from step0
     to step8 inside the level3 subgraph.
4. If a transition is expressed as a FIXED edge (not a conditional edge routing function),
   you cannot wrap it without modifying the subgraph topology. In that case:
   - Do not wrap it.
   - Add a comment in pipeline_builder.py: `# RV: pre_analysis->step0 is a fixed edge; @verify_transition not applicable.`
   - Report the finding to the squad lead.

Do not invent routing functions or split fixed edges into conditional edges to
accommodate the wiring. The non-invasive constraint takes absolute precedence.
